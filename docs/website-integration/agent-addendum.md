# Agent Addendum: Personal Website Integration

Read this after `AGENTS.md` and `docs/future-development-plan.md`.

## Product direction

This repository should become a personal static website that contains the CIM Library as a first-class project module. The atlas should remain useful on its own, but the website shell should make it discoverable from a homepage, project index, and project landing page.

## Constraints

- Keep the site static.
- Keep `/library/` and `/papers/[slug]/` stable.
- Do not add a backend, database, authentication, PDF hosting, or server runtime.
- Do not introduce coverage scores, quality scores, ranking scores, or `trajectory_IR_relevance`.
- Do not invent paper facts, artifact links, licenses, venues, years, or reproducibility claims.
- Prefer small, inspectable UI and metadata changes.

## Source of truth

- Paper metadata: `src/content/papers/*.md` frontmatter.
- Taxonomy: `src/data/taxonomy.json`.
- Website project cards: `src/data/project-registry.json`.
- Static atlas export: `public/cim-library.manifest.json` generated by `npm run export:atlas`.

## Required checks

Before reporting completion, run:

```bash
npm run qa
npm run validate
npm run export:atlas
npm run contract:website
npm run check
npm run build
```

If dependencies are not installed yet, run:

```bash
npm run smoke:website -- --fast
```

Then report which checks were skipped and why.
