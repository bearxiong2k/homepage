Core finding

The field has mature frontends, mapping/search frameworks, and simulator/ISA backends. The unresolved abstraction is the space-time path of a value through the CIM fabric: bit-sliced partial sums, analog/digital transitions, ADC/DAC sharing, digital accumulation, reconstruction, and downstream reduction.

The dominant stack shape

Across surveyed CIM/PIM compiler stacks, the front and back are relatively stable; the unstable and conceptually thin region is the middle layer.

frontend
Static compute graph

ONNX, TorchScript, linalg, TOSA, operator DAGs, or tensor expressions.

  • Expressively adequate for static workloads
  • Usually not the source of the CIM novelty
→
contested ground
Mapping / search / lowering

Layer partitioning, loop scheduling, array binding, mode selection, bit layout, cost models, hidden state, and codegen templates.

  • Most variation lives here
  • Most named “IRs” do not close over their own semantics
→
backend
ILA-ish simulator / ISA trace

Latency, energy, area, accuracy, instruction streams, command traces, or hardware drivers.

  • Useful evaluation boundary
  • Too late for high-level trajectory rewrites

The hidden-IR pathology

The nominal IR in a paper is often not the artifact that actually carries semantics.

Named artifact

Graph IR, MLIR dialect, hardware-resource abstraction, meta-op flow, instruction stream, architecture JSON, YAML config.

≠

Actual compiler semantics

Config YAML + mutated graph attributes + search-state structs + cost-model LUTs + pass ordering + codegen string templates.

Diagnostic question: Is there a single artifact that upstream passes can read, verify, and rewrite without consulting hidden state?

The trajectory-IR gap

The candidate contribution is to promote the thing many papers call “dataflow” or “path” into a typed, rewritable IR object.

Tensor value

Logical output of Conv / GEMM / MVM / attention block.

Bit-sliced placement

Columns, rows, cell precision, significance offsets.

Analog partial sum

Charge/current/voltage accumulation in array.

ADC / sensing

Shared converter, precision, rate, nonideality.

Digital partial sum

Accumulator, channel rate, local buffer lineage.

Reconstruction

Shift-add tree, significance merge, precision growth.

Downstream reducer

Activation, reduction, softmax, accumulation, storage.

Trajectory rewrite target: can a bit-sliced partial-sum trajectory be legally fused, rerouted, retimed, or consumed by a downstream reducer without reconstructing a full-precision tensor first?

Received pipeline

crossbar partial sums→reconstruction tree→full-precision value→downstream op

Reconstruction is treated as inevitable and opaque.

Trajectory-aware pipeline

typed bit-slice streams→rate/precision/domain coercions→fused reducer

The path is named, typed, checked, and rewritable.

What the field expresses well — and misses

The strengths are real: hardware hierarchy, operator binding, dense scheduling, cost-driven mapping, and backend trace generation. The misses come from not treating value flow as an IR object.
